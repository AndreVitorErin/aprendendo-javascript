História do JavaScript e sua Principais Funcionalidades

Introdução

Para conhecermos bem qualquer linguagem de programação, é importante conhecer sua história, seu propósito para que possamos entender melhor para onde essa tecnologia está indo, e quais são as possibilidades que ela oferece.

A história do JavaScript está realcionada com a história da internet, que já tratamos anteriormente nos estudo de HTML e CSS, para se aprofundar no assunto, clique aqui. Nesse artigo vamos abordar apenas os aspectos mais importante para compreensão dessa línguagem de programação tão poderosa, que é o JavaScript.

Primeiramente, é importante diferenciarmos o que é Cliente e o que Servidor.

O termo cliente refere-se ás máquinas relacionadas ao usuário, aquelas que recebem os dados, que utilizam um serviço que está sendo oferecido por um servidor. Seu computador pessoal, seu celular, tablet, etc são exemplos de clientes.

Já o servidor são máquinas cujo foco é o armazenamento de arquivos que são enviados para o cliente. E, de modo geral, que faz essa ponto é o navegador, que por sua vez, interpreta arquivos HTML (claro que existem muito mais elementos envolvidos no processo, essa é uma simplificação para compreendermos alguns conceitos básicos).

O JavaScript surgiu como uma tecnologia voltada para os clientes (client side), embora hoje ele funcione também para servidores. Programadores que desenvolvem a parte que o cliente interage são chamados de desenvolvedores Front End, e aqueles que desenvolver a parte mais relacionada ao servidor, são os desenvolvedores Back End.

Geralmente, um website é composto basicamente por três tecnologias:

  • HTML - responsável pelo conteúdo (semântica)
  • CSS - responsavél pela formatação (designe)
  • JavaScript - responsavél pela interatividade (programação)

A História da Linguagem

A internet surgiu durante a Guerra Fria e seu principal uso e interesse era militar. Gradativamente começou a ter um maior uso acadêmico, e por fim, migrou para a instância comercial.

Em 1993, em Genebra, Timothy Berners-Lee criou a linguagem de marcação HTML, o protocolo HTTP e a world wide web. Mais ou menos na mesma época, Mark Andersen criava o primeiro navegador, chamado moisaic, que era compatível com a nova linguagem criada. No ano seguinte, em 1994, Mark se une a Jim clark, e fundam a empresa netscape, que cria um navegador homônimo, que se tornou muito popular.

Já em 1995, Brendan Eich, cria uma linguagem de programação chamada moccha (nome de um estilo de café) que tinha por objetivo fornecer mais funcionalidade às páginas HTML, que até então era muito estática. Pouco tempo depois, o nome da linguagem foi modificado e passou a se chamar LiveScript

Na mesma época, a Sun Microsystem criava a linguagem Java, que ganhou grande desta na mídia, e era chamada de "linguagem do futuro". Por uma questão de marketing, a LiveScript mudou seu nome para JavaScript. Mas essas duas linguagens são bem diferentes uma da outra, e o pouco que possuem em comum, se deve ao fato de ambas serem "herdeiras" da linguagem C.

Nesse meio tempo, a Microsoft comrpou a empresa proprietária do código do navegador "mosaic", e baseado nele, lançou seu próprio navegador: o Internet Explorer. Ele foi lançada junto com o Windows 95 e já vinha instalado como padrão, o que fez o navegador netscape perder seu protagonismo, e posteriormente, a empresa foi a falência.

Vendo o sucesso do JavaScript, a Microsoft lançou sua própria linguagem de programação voltada para web, que tinha as mesmas funcionalidades, e inclusive corrigia algumas falhas. Essa linguagem foi chamada de JScript.

Em 1997, para evitar confusões que poderiam surgir devido ao lançamento da JScript, a NetScape (antes de falir, obviamente) decide padronizar a linguagem JavaScript. Essa padronização foi feita através da ECMA (uma empresa muito parecida com a ISO amerecina, mas que atua na Europa). Disso surge a linguagem ECMAScript, que nada mais é do que a linguagem JavaScript padronizada.

Em 2002, o pessoal que trabalhava na antiga NetScape, decide se unir e criam uma fundação chamada Mozilla, responsavél pelo desenvolvimento do navegador FireFox, que teve um certo protagonismo por um tempo, e ainda é bastante usado até os dias atuais.

Em 2008, a gigante Google lança seu próprio navegador, o Chrome, que em 2 anos já era o terceiro navegador mais utilizado, e atualmente (escrevo de 2020) reina soberano, tendo mais de 50% de market share.

Em 2009, o Chrome lança o V8, seu interpretador JavaScript, também chamado de motor JavaScript, que é um software especializado em interpretar e executar JavaScript ou ECMAScript. Todo navegador tem seu próprio motor:

  • FireFox - SpiderMonkey
  • Safari - Nitro
  • Opera - Carakan
  • Edge - Chakra

Porém, o V8 foi muito importante. Por ter seu código aberto, um grupo utilizou ele como base para criar o node.js. Até então, os motores JavaScript só rodavam em nevegadores, mas o node.js rodava JavaScript diretamente na máquina, o que permitiu serem exploradas novas funcionalidade, e por isso hoje é possível usar JS do lado do servidor (Back End).

A evolução do JS proporcionou a criação de diversas tecnologias que facilitam e otimizam sua aplicação (frameworks, SDK, conjunto de bibliotecas). Entre as mais relevantes temos:

  • JQuery
  • Angula (criado e mantida pelo Google - facilita a xriação de aplicações web. Utiliza uma linguagem menos imperativa e mais declarativa)
  • React (criado pelo facebook - semelhante ao Angula em muitos aspectos)
  • Vue.js (criado por um ex-programador do google que disse estar cansado do Angula, e de fato o Vue traz algumas novidades interessantes)
  • Electron (mantido pela GitHub - especializado na criação de interfaces gráficas - o próprio Visual Studio Code foi feito em Eletron)
  • Ionic (focado na criação de apps para dispositivos móveis)
  • Cordova

O JS também é utilizado para criação de jogos. As ferramentas mais usadas são:

  • Phaser
  • Pixi js
  • Impact
  • Melon.js
  • Crafty.js
