DOM - Document Object Model

DOM - Document Object Model, ou Modelo de Objetos para Documentos. Basicamente é um conjunto de objetos dentro do seu navegador que vai dar acesso aos compenetes internos do seu website. Por isso, o DOM não funciona dentro do node.js, por exemplo. Ele representa uma árvore de objetos em memória, ou seja, mapeia documentos HTML em objetos.

Árvore DOM

É muito importante fazer a áravore DOM do seu site. Ela deve começar pela raiz, que dentro do navegador chamamos de "window", então tudo do JavaScript está dentro de um objeto chamado "window", que é a janela do seu navegador.

Dentro do window temos vários outros objetos, por exemplo:

  • location - diz qual a localização do seu site (url, página atual, qual foi a página anterior, etc)
  • document - documento atual
  • history - vai guardando de onde você veio e para onde você vai, o que facilita a navegação dentro do seu site

Dentro do "document" tem um outro objeto muito importante que é o "html", e dentro do html temos dois objetos importantes, que é o "head" e o "body". Nesse caso podemos dizer que html é parent de body e head, ou que que body é child de html. Dentro de head temos "meta" e title", dentro de body temos, por exemplo, " h1, p, div e assim por diante

Navegando pelo DOM podemos criar os comandos que irão alterar o código html. A seleção de qual elemento será alterado pode ocorrer da seguinte forma:

  • por Marca - linha de comando getElementsebyTagName() - seleciona todos os objetos de uma mesma tag a não ser que haja uma especificação, que é feita usando colchetes
  • por ID - linha de comando getElementbyId()
  • por Nome - linha de comando getElementsbyName()
  • por Classe - linha de comando getElementsbyClassName()
  • por Seletor (recurso mais recente que não está disponível em todas as versões do ECMAScript) - linha de comando querySelector() ou querySelectorAll()

A criação de todos esse parâmetros é algo muito demorado, e por isso, atualmente temos frameworks que realizam essa tarefa repetitiva, permitindo que o programador se ocupe de outras atividades que exigem o raciocínio humano.

Quando ao modo de seleção dos elementos, atualmente o mais recomendado é o querySelector, que funciona de 3 formas:

  • Elemento - seleciona todos os elementos de determinado tipo. Ex: querySelector('p') irá selecionar o primeiro parágrafos
  • Classe - seleciona o primeiro elementos que perteça uma classe predeterminada. Ex: querySelector('.minhaclasse'). Nesse caso é necessário colocar o ponto (.) seguido pelo nome da classe (Class)
  • ID - seleciona um elemento que possui uma identificação específica. Ex: querySelector('#meuId'). Nesse caso é necessário colocar a hashtag (#) seguida pelo nome da identificação (Id)

Para selecionar todos os elementos de um determinado tipo ou de uma determinada classe, devemos utilizar o comando querySelectorAll.

Eventos DOM

Representa o "quando" das aplicações. É algo que acontece após uma determinada ocorrência, mas diferente das estruturas condicionais, não sabemos quando esse isto irá ocorrer. Podemos dizer que um evento reage a uma ação, que é um princípio da programação reativa.

  • mouseenter - evento disparado quando o mouse entra no elemento
  • mousemove - evento disparado quando o mouse está se movendo pelo elemento, e enquanto o mouse continuar esse movimento, o evento continuará sendo disparado
  • mousedown - evento disparado quando apertamos o botão do mouse sobre o elemento, o evento continuará sendo disparado enquanto o botão permanecer apertado
  • mouseup - evento disparado quando soltamos o botão do mouse
  • click - evento disparado quando apertamos e soltamos o botão rapidamente sobre o elemento
  • mouseout - evento disparado quando o mouse sai do elemento
  • onmouseover - mouse passou sobre o elemento
  • onload - após o carregamento da página
  • onchange - o conteúdo do elemento é alterado
  • onfocus - o elemento recebe foco
  • onblur - o elemento perder o foco
  • onselect - o elemento é selecionado (mais usado em checkboxes)
  • onsubmit - dados do formulário são enviados ao servidos
  • onkeydown - tecla pressionada
  • onkeypress - tecla pressionada e solta
  • onkeyup - tecla solta

Esses foram alguns exemplos simples. A lista de eventos possíveis é enorme, e nem todos os elementos são compatívies com todos os eventos, alguns são específicos. Para conhecer todos os eventos o recomendável é procurar diretamente pelas listas de referências.

Com a popularização dos aparelhos portáteis, hoje temos, por exemplos os touchs events, que são diferentes dos mouses events.

Para entender melhor eventos, antes é necessário entender o que é uma função, que é um conjunto de linhas de códigos que só serão executados quando um evento ocorrer. Esse conjuntos de linhas é chamado de bloco

Os blocos em JS são reconhecidos por serem escritos entre chaves ({}). Para que esse bloco seja reconhecido como uma função, ele deve ser nomeado com o nome function + a ação que será realizada. A estrutura ficará dessa forma:

function nome-ação(){
}

Dentro do parênteses que vem após o nome poderá ou não ser colocado algum parâmetro.

Quando começamos a incerir eventos e códigos mais complexos é comum que ocorram erros. Então caso você abra seu programa no navegador para testar e ele não execute corretamente, clique com o botão direito do mouse dentro do navegador, escolha a opção "inspecionar". Essa opção irá abrir a dev tools, e no menu superior, no canto esquerto, vai aparecer a sinalização de um erro.Clicando no x em vermelho, que representa o erro, abrirá a descrição desse erro, e em qual linha ele ocorreu. A partir disso, você deve verificar dessa linha pra cima.

Propriedade style

Assim como modificamos o documento HTML através dos comandos do DOM, também é possível modificar o CSS diretamente através da propriedade style. Entretando, esta não é uma boa prática, pois não é interessante injetarmos código CSS em nosso JavaScript. Uma boa prática seria a adição e remoção de classes.

Para isso, utilizamos os comandos classList.add() e classeList.remove(), sendo que um adiciona e outro remove um elemento da lista.

Evitando envio de dados ao servidor

Por padrão, ao clicar em um botão do tipo submit de um formulário HTML os dados são enviados ao servidor e a página é recarregada.

Atualmente, grande parte dos sistemas web funciona como SPA - Single Page Application, deste modo, deve-se evitar o refresh com event.preventDefault. Nesse modelo, os dados reagem às interações do usuário instantaneamente.
